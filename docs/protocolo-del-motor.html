Protocolo del motor — comandos texto stdin/stdout

← Inicio · Arquitectura del Sistema · Arquitectura API

⚠ Incompleto respecto al código (revisado 2026-06-27)

Esta ficha solo cubre los 4 comandos originales (IGNITE, TICK, RESET, QUIT). El motor real (engine/main.cpp) reconoce además: FIREBREAK_BATCH, FIREBREAK_UNDO_BATCH, WIND, SMOKE_VIRTUAL, QUADRANTS, FUEL_MAP, STATS, PHYSICS_MODE, SURFACE_LAYERS y MODE, con sus respuestas (FIREBREAK_DONE, FIREBREAK_UNDO_DONE, QUADRANTS …, etc.). El enum STATE de los CHANGED incluye también CORTAFUEGOS (se emite) y EMPTY (no se emite). Contrato de cortafuegos en Contrato de Cortafuegos (auditoría M2).

Por qué texto y no JSON

El motor monolítico anterior se comunicaba con la API enviando el grid completo (100.000 celdas) serializado como JSON en cada tick. El rediseño abandona ese esquema por dos razones:

  1. Eficiencia: solo unas pocas miles de celdas cambian entre ticks. JSON completo es órdenes de magnitud más grande de lo necesario.
  2. Simplicidad: el motor no necesita parser JSON ni la API necesita serializar todo el estado. Un comando por línea es trivial de leer y escribir en C++ y en Python.

El protocolo es texto plano, una orden por línea:

  • API → motor: comandos IGNITE, TICK, RESET, QUIT.
  • motor → API: líneas CHANGED ... y TICK_DONE N.

Comandos API → motor

IGNITE wx wz

Enciende la celda de superficie en las coordenadas mundiales (wx, wz) en metros. Activa el cuadrante correspondiente si aún no lo estaba.

IGNITE 275 275
IGNITE 299.5 275.5     ← coordenadas float también funcionan

No genera respuesta inmediata.

TICK

Avanza un tick. Procesa todos los cuadrantes activos y emite las celdas que hayan cambiado, terminando con TICK_DONE. Ver pipeline: Pipeline de Simulación.

RESET

Limpia el estado dinámico del motor: desactiva todos los cuadrantes y libera sus SubGrids. Tras RESET, el siguiente TICK no genera celdas.

QUIT

Cierra el proceso limpiamente. La API lo lanza al apagarse.

Mensajes motor → API

CHANGED wx wy wz STATE TEMP

Una línea por cada celda que ha cambiado de estado o cuya temperatura ha variado más de UMBRAL_TEMP_CHANGED = 0.5 °C respecto al último valor emitido para esa celda.

CHANGED 275 4 275 BURNING 800
CHANGED 276 4 275 FUEL 25
CHANGED 275 5 275 AIR 250.5
Campo Tipo Rango
wx int [0, 1598) — metro mundial X
wy int [0, 20) — nivel vertical
wz int [0, 1393) — metro mundial Z
STATE enum AIR, FUEL, BURNING, SMOKE
TEMP float grados Celsius

El motor mantiene un caché por celda con el último (state, temp) emitido y solo escribe CHANGED cuando alguno cambia significativamente.

TICK_DONE N

Cierra el bloque de un TICK. N es el número total de celdas procesadas en ese tick (suma de SubGrid::total = 50.000 por cada cuadrante activo).

TICK_DONE 0          ← sin cuadrantes activos (estado inicial o tras RESET)
TICK_DONE 50000      ← 1 cuadrante activo
TICK_DONE 100000     ← 2 cuadrantes (el fuego cruzó una frontera)
TICK_DONE 450000     ← 9 cuadrantes (incendio plenamente extendido)

TICK_DONE permite a la API saber dónde acaba la respuesta de un tick sin contar líneas ni esperar a un timeout.

Ejemplo de sesión

$ engine/motor data/world.dat
IGNITE 275 275
TICK
CHANGED 250 4 250 FUEL 25
CHANGED 251 4 250 FUEL 25
... (≈2.500 líneas de FUEL al activar el cuadrante)
CHANGED 275 4 275 BURNING 800
TICK_DONE 50000
TICK
CHANGED 274 4 275 BURNING 142
... (propagación a vecinos)
TICK_DONE 50000
RESET
TICK
TICK_DONE 0
QUIT

Implementación

  • C++: engine/main.cpp. Parser por getline + istringstream, emisión por std::cout con flush al final de cada TICK_DONE.
  • Python: ver Arquitectura API sección motor.py. El wrapper motor.tick() lee líneas hasta TICK_DONE y devuelve la lista de cambios parseados.
