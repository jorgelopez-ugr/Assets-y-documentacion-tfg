Error: SubViewportContainer dentro de CanvasLayer no renderiza

← Inicio · Integracion Godot-Motor — Recopilatorio

Milestone afectado: Integración Godot-Motor (#80, veleta del viento) Archivos involucrados: godotUI/simulacion.gd

El síntoma

La veleta 3D del viento (#80) se diseñó como widget HUD: un SubViewport con su propio World3D + Camera3D + flecha 3D que rota según el viento. Mostrada al usuario vía un TextureRect anclado a la esquina superior derecha.

Primer intento:

veleta_sub_viewport = SubViewport.new()
veleta_sub_viewport.own_world_3d = true
add_child(veleta_sub_viewport)              # hijo del Node3D root
# ... añadir flecha + cámara + luz al SubViewport ...

# En _setup_hud:
veleta_hud_rect = TextureRect.new()
veleta_hud_rect.texture = veleta_sub_viewport.get_texture()
veleta_hud_rect.set_anchors_preset(Control.PRESET_TOP_RIGHT)
veleta_hud_rect.position = Vector2(-200, 20)
hud.add_child(veleta_hud_rect)              # hijo del CanvasLayer

No se veía nada en la esquina. El usuario reportó:

"La flecha es inexistente. Vamos a intentar arreglar eso."

Por qué pasaba (dos problemas combinados)

1. ViewportTexture cruzando subárboles

Cuando un TextureRect consume SubViewport.get_texture(), internamente guarda un ViewportTexture con la ruta relativa al SubViewport fuente. Si el TextureRect y el SubViewport viven en subárboles distintos (Node3D root vs CanvasLayer del HUD), Godot no resuelve la ruta correctamente y la textura queda vacía.

Es comportamiento documentado en la docs de Godot pero no obvio.

2. CanvasLayer no propaga anchors

Aunque resolviéramos el bug anterior, los Control colgados directamente del CanvasLayer no tienen un rect padre coherente contra el que resolver set_anchors_preset(PRESET_TOP_RIGHT). El preset apunta a la esquina (1, 0), pero "uno" relativo a nada da posición ambigua.

Resultado: el TextureRect quedaba en posición (0, 0) con tamaño posiblemente cero, sin renderizar.

La solución (dos partes)

a) SubViewportContainer en lugar de TextureRect + ViewportTexture

El SubViewportContainer es un Control específico de Godot 4 diseñado para envolver SubViewports. Resuelve internamente la textura sin requerir paths cruzados:

veleta_container = SubViewportContainer.new()
veleta_container.stretch = true
veleta_container.size = VELETA_HUD_TAM

veleta_sub_viewport = SubViewport.new()
veleta_sub_viewport.world_3d = World3D.new()    # explícito
veleta_container.add_child(veleta_sub_viewport)  # SubViewport hijo del Container

Aprendizaje secundario: own_world_3d = true es lazy; mejor asignar world_3d = World3D.new() directamente para garantizar que el mundo está listo en el primer frame.

b) Wrapper Control con PRESET_FULL_RECT en el CanvasLayer

Para que los anchors de hijos funcionen:

hud = CanvasLayer.new()
add_child(hud)

pantalla_hud = Control.new()
pantalla_hud.set_anchors_and_offsets_preset(Control.PRESET_FULL_RECT)
pantalla_hud.mouse_filter = Control.MOUSE_FILTER_IGNORE
hud.add_child(pantalla_hud)

# Ahora pantalla_hud SÍ tiene un rect coherente. Los hijos pueden
# usar PRESET_TOP_RIGHT, PRESET_BOTTOM_LEFT, etc. y se resuelven
# contra el viewport principal.
pantalla_hud.add_child(veleta_container)

c) Cámara del SubViewport con current = true

Detalle final que aprendí: una Camera3D dentro de un SubViewport con own_world_3d=true no se hace current automáticamente. Hay que llamar cam.current = true después de añadirla al árbol (porque make_current() necesita estar en la jerarquía para registrarse en su Viewport).

var cam_widget := Camera3D.new()
veleta_sub_viewport.add_child(cam_widget)        # primero al árbol
cam_widget.global_position = Vector3(0, 18, 28)  # luego look_at
cam_widget.look_at(Vector3.ZERO, Vector3.UP)
cam_widget.current = true                        # registra como current

Resultado

Tras los tres cambios, la veleta se ve en la esquina superior derecha, rotando con el viento y escalando con la magnitud, desde cualquier cámara del mundo (las 6 isométricas y la libre).

Lección general

En Godot 4, CanvasLayer no es un Control. Cualquier UI que requiera anchors necesita un Control wrapper con PRESET_FULL_RECT entre el CanvasLayer y los demás Controls.

Para mostrar un SubViewport como widget en una UI, usar SubViewportContainer siempre. El path explícito con ViewportTexture puede fallar silenciosamente cuando el SubViewport y el consumidor viven en subárboles distintos.

Ambas reglas son específicas de Godot 4 y no totalmente obvias en la documentación. Documentadas aquí para no repetir el ciclo en la fase de "HUD usable" del próximo milestone.
