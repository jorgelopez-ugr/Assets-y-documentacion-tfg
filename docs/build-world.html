⚠ Sección de `fuel` desactualizada (revisado 2026-06-27)

El fuel_coefficient ya no es 1.0 uniforme: se deriva de la clasificación hiperespectral (capas de especies, alineadas por homografía SIFT en #139, vía alinear_hiperespectral.clasificar()). Por tanto el apartado "Lo que aún no hace → No usa imagen hiperespectral", la fila "fuel = 1.0 dentro del DEM" y los tests que asumen fuel uniforme están obsoletos. Mapeo real en build-world-leyendo-especies. El resto (header, cuadrantes, formato binario, recorte de bordes) sigue vigente.

Archivo fuente: tools/build_world.py Tests: tools/test_build_world.py (13 asserts) Genera: data/world.dat (binario, ~11 MB, versionado)

Para qué existe este script

Sustituye al antiguo build_initial_state.py. Lee el DEM real (heightmap_16bit.png) y precomputa todo lo que el motor necesita saber del terreno una sola vez. El resultado se versiona en el repositorio para que cualquiera pueda compilar el motor y arrancar la simulación sin tener que volver a generarlo.

El script se ejecuta una vez por terreno. Si el heightmap_16bit.png no cambia, no hace falta regenerarlo.

Qué precomputa

Dato Origen Almacenado en world.dat
Dimensiones del mundo constantes (1598×1393 m) header
División en cuadrantes constantes (50×50 m → 32×28) header
Niveles verticales constantes (20) header
ID de cada cuadrante índice secuencial por cuadrante
Origen mundial del cuadrante (qx*50, qz*50) por cuadrante
Vecinos (Moore, 8) grafo de adyacencia por cuadrante
Pendiente media np.gradient sobre el bloque 50×50 del DEM por cuadrante
Dirección de pendiente np.gradient (vector XZ) por cuadrante
Nivel vertical de superficie DEM remapeado a [0, 19] por celda de superficie
fuel_coefficient 1.0 dentro del DEM, 0.0 fuera por celda de superficie

Constantes y decisiones

WIDTH_METROS  = 1598
DEPTH_METROS  = 1393
CELDA_TAM_M   = 50           # tamaño de cuadrante en metros
NIVELES_Y     = 20
ELEV_MIN      = 1597.0
ELEV_MAX      = 1743.0
MAGIC         = b"WLD1"

Recorte de bordes

1598 / 50 = 31.96 y 1393 / 50 = 27.86: los cuadrantes del borde derecho/inferior no son cuadrados completos. La decisión adoptada es mantener una rejilla regular 32 × 28 = 896 cuadrantes y recortar:

  • Última columna (qx=31): cuadrantes de 48 × 50 celdas válidas.
  • Última fila (qz=27): cuadrantes de 50 × 43 celdas válidas.
  • Esquina inferior derecha (qx=31, qz=27): 48 × 43 celdas válidas.

Las celdas fuera del DEM dentro de un cuadrante recortado se rellenan con y_superficie = 0 y fuel_coefficient = 0.0 (sentinel: nunca arden, son terreno virtual).

Vecindad Moore (8)

Cada cuadrante interior tiene 8 vecinos (los 8 cuadrantes que rodean al suyo). Esquinas: 3. Bordes no esquina: 5. Esto se valida en los tests.

Mapeo elevación → nivel vertical

t = (elev_m - ELEV_MIN) / (ELEV_MAX - ELEV_MIN)
y = round(t * (NIVELES_Y - 1))     # 0..19

20 niveles cubren los 146 m de desnivel del DEM, ~7.3 m por nivel. El doc del rediseño dice "~5 m" pero se priorizó usar literalmente los 20 niveles para mantener el SubGrid pequeño.

Pendiente

Se calcula con np.gradient sobre el bloque del DEM correspondiente al cuadrante. Se guarda:

  • pendiente_media: módulo medio del gradiente.
  • pendiente_dir: dirección media (Vec2 sobre XZ).

Formato binario

Little-endian, sin padding. Ver detalle: world › Formato binario de world.dat.

HEADER_FMT      = "<4sHHBBBBB"
QUAD_HEADER_FMT = "<Ifffff"     # id + 5 floats
CELL_FMT        = "<Bf"          # y_sup + fuel

Uso

tools/.venv/bin/python tools/build_world.py
# → data/world.dat (~11 MB)

El script verifica que heightmap_16bit.png mide exactamente 1598×1393. Si no, falla con un mensaje claro.

Tests

13 asserts en tools/test_build_world.py:

  • Header correcto: magic, dims, conteos.
  • 896 cuadrantes generados, ids secuenciales.
  • origen = (qx*50, qz*50) exacto.
  • Vecinos: 3 en esquinas, 5 en bordes, 8 en interior.
  • Sin duplicados ni autorreferencias en vecinos.
  • y_superficie ∈ [0, 19].
  • Cuadrante interior tiene fuel = 1.0 en todas las celdas.
  • El total de celdas con fuel = 1.0 cuadra con 1598 × 1393 = 2.226.014 — esto demuestra que el recorte de bordes es exacto: ni una celda válida perdida, ni una sentinel contada como válida.
cd tools && .venv/bin/python -m pytest test_build_world.py -v

Sincronía con el motor

build_world.py y engine/world.h son las dos caras del mismo formato. Cualquier cambio (constantes, orden de campos, tipos) debe aplicarse a ambos y comprobarse con:

tools/.venv/bin/python tools/build_world.py    # genera
bash engine/test_world_loader.sh               # valida que el motor lo lee bien

Lo que aún no hace

  • No usa imagen hiperespectral: fuel_coefficient es 1.0 para todas las celdas del DEM. Cuando se integre, este es el lugar donde el coeficiente por especie pasaría a world.dat.
  • No calcula caminos forestales ni cortafuegos. Esos podrían entrar como un fuel_coefficient = 0.0 en columnas específicas.
