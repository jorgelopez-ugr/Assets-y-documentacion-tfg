Contrato actual de cortafuegos — auditoría (#117, M2)

Issue #117. Antes de tocar nada en M2 (terraformado), se documenta el contrato real del cortafuegos tal como está hoy, leído del código. M2 (#118/#119) y M3 (trazado automático) deben reutilizar este mismo contrato, no inventar uno nuevo. Fuente: api/main.py, api/motor.py, engine/main.cpp, engine/simulacion.h.

ℹ Foto previa a M2 — algunas partes ya cambiaron (revisado 2026-06-27)

Esta auditoría retrata el contrato antes de implementar el terraformado. Dos cosas ya no son así: al pintar con intensidad 1.0 la celda pasa a CORTAFUEGOS (no a AIR) — #118 — y el "canal de vuelta de alturas" que aquí se plantea como el cambio grande de #119 no se hizo: el terraformado acabó siendo visual en Godot. El resto del contrato (/firebreak, lote, migas, coordenadas, dualidad estática/viva) sigue vigente. Ver Terraformado de cortafuegos (M2).

Resumen en una frase

Godot manda a la API un vector de celdas [[x, z], ...] (índices de celda mundial, 1 celda = 1 m) + una intensidad ∈ [0,1]; la API lo reenvía al motor en lote; el motor reduce el fuel_coefficient de cada celda (nuevo = actual × (1 − intensidad)) y, si llega a 0, la celda de superficie deja de arder.

Capa 1 — API HTTP (api/main.py)

POST /firebreak

// request
{ "cells": [[x, z], ...],   // pares de enteros, índice de celda mundial
  "intensidad": 1.0 }        // float [0,1], default 1.0 (pydantic valida rango)
// response
{ "status": "ok", "painted": N }   // N = celdas que el motor reportó procesadas

DELETE /firebreak (deshacer)

// request (mismo formato, SIN intensidad)
{ "cells": [[x, z], ...] }
// response
{ "status": "ok", "undone": N }

DELETE con cuerpo JSON: requiere Body(...) explícito en FastAPI.

Modelos: FirebreakPayload(cells: list[tuple[int,int]], intensidad: float=1.0), FirebreakUndoPayload(cells).

Capa 2 — Wrapper del motor (api/motor.py)

  • pintar_firebreak(cells, intensidad) -> int: escribe FIREBREAK_BATCH <I> <N>
  • N líneas "x z", lee hasta FIREBREAK_DONE N, devuelve N.
  • deshacer_firebreak(cells) -> int: escribe FIREBREAK_UNDO_BATCH <N> + N líneas "x z", lee hasta FIREBREAK_UNDO_DONE N.
  • Ambas bajo _lock (serializa el acceso al proceso C++).

Capa 3 — Protocolo stdin/stdout (engine/main.cpp)

API → motor:  FIREBREAK_BATCH <intensidad> <N>\n   luego N líneas "<x> <z>\n"
              FIREBREAK_UNDO_BATCH <N>\n            luego N líneas "<x> <z>\n"
motor → API:  FIREBREAK_DONE <N>\n
              FIREBREAK_UNDO_DONE <N>\n
  • Cabecera malformada (no parsea intensidad/N, o N<0) → comando ignorado, no consume líneas.
  • Línea de celda malformada → se salta, NO cuenta como procesada.
  • procesadas = nº de líneas sintácticamente válidas (puede diferir de las que realmente cambiaron algo).

Capa 4 — Lógica del motor (engine/simulacion.h)

pintar_firebreak(wx, wz, intensidad)

  1. wx<0 || wz<0 → no-op. Fuera de la rejilla de cuadrantes → no-op.
  2. Miga (undo): si la celda no está ya en fuel_modificado_ (map key_firebreak(qid,idx) → fuel_original), guarda el valor original. Repintar NO sobrescribe la miga (preserva el original real).
  3. nuevo_fuel = fuel_coefficient[idx] × (1 − clamp(intensidad,0,1)). Modifica la copia estática Cuadrante::fuel_coefficient[idx] (sobrevive a la desactivación del cuadrante).
  4. Si el cuadrante está ACTIVO y la celda viva de superficie está en estado FUEL: propaga el cambio a la Celda. Si nuevo_fuel ≤ 0 → la celda pasa a AIR (fuel=0); si no, actualiza su fuel_coefficient.
  5. BURNING / SMOKE / AIR no se tocan ("una motosierra no apaga incendios").

deshacer_firebreak(wx, wz)

  • Sin miga (nunca se pintó / ya deshecho) → no-op.
  • Restaura fuel_coefficient[idx] al original y borra la miga.
  • Si activo: FUEL → actualiza fuel; AIR con original>0 → vuelve a FUEL; BURNING/SMOKE → no se tocan.
  • /reset (RESET) restaura todas las migas antes de desactivar (simulacion.h:270).

Hechos clave para M2/M3

  1. Coordenadas: enteros, índice de celda mundial (1 celda = 1 m). NO metros float, NO índices de cuadrante. Godot ya emite en este formato.
  2. El cortafuegos vive en dos sitios: la copia estática del cuadrante (Cuadrante::fuel_coefficient, persistente) y la celda viva (Celda, solo si el cuadrante está activo). Cualquier cambio de M2 debe respetar esta dualidad.
  3. El cortafuegos NO emite CHANGED: FIREBREAK_BATCH solo responde FIREBREAK_DONE N. Godot pinta las celdas client-side (él las envió). ⚠️ Para el terraformado (#119) esto es el punto clave: el motor tendrá que devolver las nuevas alturas de la franja a Godot (hoy no devuelve nada geométrico) — o por un campo nuevo en la respuesta, o por un endpoint/GET. Hay que decidir el mecanismo SIN romper el contrato de pintado existente.
  4. intensidad ya existe como atenuación de fuel (no binario). El "anular combustible" de #118 es el caso intensidad=1.0.
  5. El trazado automático (M3) solo tiene que producir el mismo vector [[x,z],...] y mandarlo por POST /firebreak. Cero código nuevo de pintado.

Qué le falta al contrato para M2 (terraformado)

Hoy el cortafuegos solo toca fuel_coefficient. M2 añade: - Marcar las celdas como CORTAFUEGOS (estado/flag) — decidir si nuevo estado de celda o flag aparte. - Aplanar el terreno (y_superficie) en la franja → es lo que obliga a devolver las nuevas alturas a Godot (punto 3). - Material/color distinto en Godot (#120, lado cliente).

Estas tres son #118 (consolidar anulación de fuel), #119 (terraformado motor+API) y #120 (Godot). Esta auditoría es la base de #119: el cambio grande es el canal de vuelta de alturas, que hoy no existe.
