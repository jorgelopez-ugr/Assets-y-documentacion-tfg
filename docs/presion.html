Para qué existe este módulo

Garantiza que el fluido simulado se comporte como incompresible: lo que entra en una celda debe salir. Sin esto, la advección crearía o destruiría material de la nada.

También es el mecanismo que permite la propagación del fuego. Ver: mecanismo-propagacion-via-presion

Las tres fases en orden

Grid resolverPresion(const Grid& g, int iteraciones = 30) {
    Grid resultado = iterarJacobi(g, iteraciones);  // fases 1 y 2
    corregirVelocidad(resultado);                    // fase 3
    return resultado;
}

Fase 1: calcularDivergencia

static float calcularDivergencia(const Grid& g, int x, int y, int z)

Mide si una celda "expulsa" o "acumula" fluido:

div = (vx_derecha − vx_izquierda)
    + (vy_arriba  − vy_abajo)
    + (vz_frente  − vz_atras)
  • div > 0 → el fluido se expande (sale más de lo que entra)
  • div < 0 → el fluido se comprime (entra más de lo que sale)
  • div = 0 → fluido incompresible ✓

En los bordes del grid se usa la velocidad propia como vecino (diferencia cero → gradiente cero → condición de contorno de Neumann).

Fase 2: iterarJacobi

static Grid iterarJacobi(const Grid& g, int iteraciones)

Resuelve la ecuación de Poisson ∇²p = divergencia iterativamente. No tiene solución directa en 3D, así que el método de Jacobi itera:

p_nueva = (p_der + p_izq + p_arr + p_abj + p_frt + p_atr − divergencia) / 6

Cada celda actualiza su presión como el promedio de sus 6 vecinas corregido por la divergencia local. Se repite iteraciones veces.

¿Cuántas iteraciones?

--iter Efecto
5 (producción rápida) Presión aproximada. El fluido no es perfectamente incompresible pero la simulación es rápida
20–30 (tests) Buena convergencia. Los gradientes de presión se propagan más lejos → mejor propagación lateral del fuego
100+ Convergencia casi exacta, coste alto

Con el grid 100×10×100 y iter=5, cada tick completa en ~30ms. Con iter=30, ~100ms.

Por qué Jacobi y no otro método

Jacobi es extremadamente simple y paralelizable. Para visualización en tiempo real, la precisión no necesita ser perfecta — lo importante es que el fluido "parezca" incompresible visualmente, no que lo sea matemáticamente.

Fase 3: corregirVelocidad

static void corregirVelocidad(Grid& g)

Resta el gradiente de presión al campo de velocidades:

v.x -= (p_derecha − p_izquierda) / 2
v.y -= (p_arriba  − p_abajo)     / 2
v.z -= (p_frente  − p_atras)     / 2

Esto elimina la componente "compresible" de la velocidad. El resultado es un campo que cumple ∇·v ≈ 0.

Esta corrección también es lo que da velocidad lateral a las celdas FUEL adyacentes a una celda BURNING — el gradiente de presión las empuja hacia la zona de baja presión creada por el gas que sube.

Condición de borde: presión cero en los bordes del grid

Cuando un vecino está fuera del grid, se usa pressure = 0.0 en la iteración Jacobi. Esto es equivalente a una condición de contorno de Dirichlet: la presión en el exterior es cero. El efecto práctico es que el fluido puede "escapar" por los bordes del grid libremente, lo que es correcto para un dominio abierto.
