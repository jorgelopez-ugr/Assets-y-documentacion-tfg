✎ Revisión 2026-06-27 — faltan 2 estados; `fuel` ya no es uniforme

El enum real tiene 6 estados, no 4: además de AIR/FUEL/BURNING/SMOKE están EMPTY (aire inerte en modo superficie, #112) y CORTAFUEGOS (suelo despejado, #118) — por eso la nota "AIR sustituye a EMPTY" de abajo quedó a medias: EMPTY volvió como estado del aire apagado. Además, fuel_coefficient ya no es 1.0 en todas las celdas: se deriva por celda de la clasificación hiperespectral (ver build-world-leyendo-especies). El struct Celda y la tabla de campos sí están al día.

Archivo fuente: engine/celda.h Usado en: todo el motor — es el tipo base de todas las operaciones

Para qué existe este módulo

Define los tipos de datos fundamentales del motor: EstadoCelda, Vec3, Vec2 y Celda. No contiene ninguna lógica — es puro almacenamiento. Todos los sistemas (buoyancy, presion, adveccion, reaccion) reciben y devuelven celdas de este tipo.

EstadoCelda — los cuatro estados físicos

enum class EstadoCelda {
    AIR,      // por encima de la superficie. Transporta calor y humo. No arde.
    FUEL,     // celda de superficie con combustible. Ignita a > 100 °C.
    BURNING,  // ardiendo activamente. Emite calor sostenido.
    SMOKE     // combustible agotado. Sigue transportando.
};
✎ AIR sustituye a EMPTY

El motor anterior tenía EMPTY como cuarto estado (sin material). En el rediseño se sustituyó por AIR porque toda celda dentro de un cuadrante activo representa volumen físico real — sobre la superficie hay aire, no "nada". El cambio no es solo cosmético: AIR participa en advección y transporte de humo, EMPTY no debía hacerlo.

Transiciones:

AIR  ──(siempre)──▶  AIR
FUEL ──(temp > 100 °C)──▶  BURNING ──(reaction → 0)──▶  SMOKE

SMOKE no vuelve a BURNING: no hay re-ignición.

Vec3 — velocidad del fluido

struct Vec3 {
    float x = 0.0f;
    float y = 0.0f;
    float z = 0.0f;
};

Velocidad del fluido en una celda. Z es el eje vertical en la convención del motor (la flotabilidad escribe en velocity.z). X e Y son horizontales.

Cuidado con la convención: el SubGrid usa sizeX/sizeY como ejes horizontales (world X, world Z) y sizeZ como vertical. La elección histórica viene del motor monolítico anterior y se mantuvo en el rediseño para no tocar la física. Ver: Cuadrante y Celda sección "Mapeo de ejes".

Vec2 — vector horizontal

struct Vec2 {
    float x = 0.0f;
    float z = 0.0f;
};

Solo dos componentes (X, Z) porque se usa para datos 2D del terreno: origen del cuadrante en metros mundiales, dirección de pendiente, etc.

Celda — la unidad de simulación

struct Celda {
    Vec3  velocity;
    float pressure         = 0.0f;
    float density          = 0.0f;
    float temperature      = 0.0f;
    float reaction         = 0.0f;
    float fuel_coefficient = 1.0f;
    EstadoCelda state      = EstadoCelda::AIR;
};

Qué hace cada campo

Campo Quién lo escribe Quién lo lee
velocity buoyancy, presion adveccion
pressure presion presion
density inicial, adveccion buoyancy
temperature inicial, adveccion, simulacion::sostener_burning buoyancy, reaccion, simulacion::evaluar_activacion
reaction inicial, reaccion reaccion, adveccion
fuel_coefficient simulacion::activar (lee del Cuadrante) reaccion
state simulacion::activar, reaccion todos los sistemas

Semántica de reaction según el estado

El campo reaction (Y ∈ [0, 1]) tiene significado distinto según el estado:

Estado reaction
AIR 0.0 — no aplica
FUEL 0.0 — sin combustión activa
BURNING 1.0 al encenderse, decrece hasta 0
SMOKE 0.0 — combustible agotado

Ver reaction-campo-fuel-vs-burning.

fuel_coefficient — para el futuro mapa hiperespectral

Escala la velocidad de consumo: reaction -= k * dt * fuel_coefficient.

  • 1.0 → combustión normal (valor actual de todas las celdas)
  • 0.0 → nunca arde (rocas, agua, cortafuegos, celdas fuera del DEM en cuadrantes recortados de borde)
  • 0.3 → arde lento (madera húmeda, futuro)

En el SubGrid del cuadrante, cada celda tiene su fuel_coefficient propio (copiado desde el array de superficie del Cuadrante en el momento de activar). Cuando se integre la imagen hiperespectral, tools/build_world.py escribirá los valores reales por especie en world.dat.

Memoria por celda

Pieza Bytes
Vec3 velocity 12
5 floats (pressure, density, temperature, reaction, fuel_coefficient) 20
EstadoCelda state (enum int) 4
Total 36

Un SubGrid de 50×50×20 = 50.000 celdas ocupa ~1.8 MB. Durante advección se mantienen 3 SubGrids simultáneos (adelante, atrás, resultado): pico de ~5 MB por cuadrante activo.
