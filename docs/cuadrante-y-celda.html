El Grid y la Celda — estructuras de datos fundamentales

← Inicio · Arquitectura del Sistema

⚠ Documento desactualizado (revisado 2026-06-27)

Esta ficha describe el motor monolítico previo al rediseño por cuadrantes: la Celda que muestra no tiene fuel_coefficient, su estado por defecto es EMPTY y el ciclo es EMPTY → BURNING → SMOKE (modelo viejo, sin FUEL). El motor real usa 6 estados (AIR/FUEL/BURNING/SMOKE/EMPTY/CORTAFUEGOS), el grid vive en engine/cuadrante.h como SubGrid de 50×50×20 (no existe engine/grid.h, ni el 100×10×100 de aquí) y el ciclo real es FUEL → BURNING → SMOKE. Para el estado vigente ver celda, cuadrante y simulacion.

La Celda (Celda)

Archivo: engine/celda.h

Una celda es el átomo del sistema. Representa un volumen cúbico del espacio de simulación y almacena el estado físico del fluido en ese punto.

struct Celda {
    Vec3  velocity;             // dirección y velocidad del fluido (m/s)
    float pressure    = 0.0f;  // presión del fluido (Pa)
    float density     = 0.0f;  // densidad del humo (kg/m³)
    float temperature = 0.0f;  // temperatura (°C)
    float reaction    = 0.0f;  // combustible: 1.0 = lleno, 0.0 = agotado

    EstadoCelda state = EstadoCelda::EMPTY;
};

Por qué estos campos y no otros

velocity — el campo de velocidades es el "motor" del fluido. Sin él, el fuego no podría moverse ni el humo ascender. Es un Vec3 porque el fluido se mueve en las tres dimensiones del espacio.

pressure — la presión no es un campo físico observable directamente, sino un campo auxiliar que usamos para forzar la incompresibilidad del fluido. Ver Pipeline de Simulación paso 2.

density — la densidad del humo en cada celda. Cuanta más densidad, más opaca visualmente debería ser la celda. También modera la flotabilidad (humo denso sube menos que humo ligero).

temperature — la temperatura impulsa la flotabilidad y es el campo visual principal para representar el fuego. A 800 °C una celda está ardiendo activamente; a 300 °C es humo caliente; a 20 °C es aire ambiente.

reaction — la "coordenada de reacción" del paper de Nvidia. Representa cuánto combustible queda. Es un valor normalizado [0, 1] que simplifica enormemente el modelo: no necesitamos rastrear el tipo de material, solo cuánto queda por quemar.

Los tres estados de una celda

EMPTY   → sin material significativo, ignorada en la serialización
BURNING → ardiendo, tiene combustible (reaction > 0)
SMOKE   → combustible agotado, sigue transportando temperatura y densidad

La transición siempre es en una dirección: EMPTY → BURNING → SMOKE. Una celda nunca vuelve a BURNING desde SMOKE (no hay re-ignición en este MVP).

El Grid

Archivo: engine/grid.h

El grid es un cubo de celdas en memoria. Sus dimensiones son sizeX × sizeY × sizeZ.

Almacenamiento lineal en memoria

Las celdas se almacenan en un std::vector<Celda> contiguo en memoria (caché-friendly). La conversión de coordenadas 3D a índice lineal usa esta fórmula:

index(x, y, z) = x + sizeX * (y + sizeY * z)

Esto equivale a "primero varía x, luego y, luego z" — orden row-major en z.

Por qué importa el orden: Los bucles del motor siempre iteran z → y → x (el eje que varía más rápido va en el bucle más interno). Esto garantiza acceso secuencial a memoria, minimizando los fallos de caché.

La función isValid

bool isValid(int x, int y, int z) const

Antes de acceder a un vecino, siempre se comprueba si está dentro del grid. Los vecinos fuera del borde se tratan de forma conservadora: presión 0, velocidad igual a la celda actual (gradiente cero en los bordes).

Esto implementa condiciones de contorno de tipo "pared": el fluido no sale del grid. Es una aproximación válida para MVP 3; en iteraciones futuras se podrían implementar condiciones de contorno abiertas (el humo "escapa" por los bordes superiores).

Tamaño actual y escalabilidad

100 × 10 × 100 = 100.000 celdas
Cada celda ≈ 7 floats + 1 enum = ~32 bytes
Total en memoria: ~3.2 MB por grid

Durante un tick se manejan hasta 3 grids simultáneos (original, adelante, atrás en la advección MacCormack) → pico de ~10 MB. Ver: Pipeline de Simulación · Protocolo del Motor

A 15 ticks/segundo, el motor procesa ~1.5M celdas/segundo. Es suficiente para visualización en tiempo real en MVP 3.
