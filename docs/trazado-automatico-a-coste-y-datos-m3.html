Trazado automático A* — coste y datos (#121, M3)

Cómo se calcula un cortafuegos automático y con qué datos. El A* vive en Python (el pathfinding no es física → fuera del motor C++). Acompaña a Contrato de Cortafuegos (auditoría M2).

Idea

Dado un origen y un destino en metros reales, el A busca el camino que minimiza un coste ponderado sobre la rejilla de celdas (1 celda = 1 m, 8 vecinos) y devuelve un vector de celdas [[x,z],…]*, idéntico al del dibujado manual. Ese vector se enviará por el flujo de /firebreak (#122) y reutiliza todo lo de M2 (pintado, terraformado, estado CORTAFUEGOS, enfriamiento) sin código nuevo.

La función de coste del trazado (a priori) es distinta de la evaluación (a posteriori, según cómo se comporta el fuego, M4): el A* propone el camino más barato; la simulación valida si contiene bien el incendio.

Función de coste (parametrizable)

Coste de entrar en una celda (x,z):

w_longitud  · paso            (1 recto, √2 diagonal)
w_pendiente · pendiente(x,z)  (m/m)
w_biomasa   · fuel(x,z)       (0..1)
w_especies  · 1{especie penalizada}

Los cuatro pesos (CosteParams) son ajustables — la propuesta del TFG pide análisis de sensibilidad. Defaults de partida: w_longitud=1, w_pendiente=5, w_biomasa=2, w_especies=5 (a afinar; "los costes se pueden tocar más tarde").

Especies penalizadas por defecto (decidido con el usuario): pino carrasco (cian), encina (violeta) y quejigo (naranja) — se preservan los árboles. El matorral y el resto NO se penalizan (el matorral es de hecho buen sitio para un cortafuegos).

Datos (todos finos, por celda)

  • Pendiente / elevación: dem.py lee heightmap_16bit.png (1 píxel = 1 m), misma convención de elevación que build_world.py → coincide con world.dat. La pendiente es la magnitud del gradiente (m/m).
  • Fuel: cargar_fuel() parsea world.dat en Python (formato documentado) → matriz por celda. Misma fuente que usa la simulación.
  • Especies: cargar_especies_penal() lee data/especies_map.png (color por especie, lo genera build_world) → máscara booleana de celdas penalizadas. Si el PNG no existe, máscara vacía (el trazado sigue con los otros factores).

A* (implementación)

  • Rejilla de 8 vecinos; heapq; heurística = distancia euclídea × w_longitud (cota inferior admisible → A* óptimo).
  • Entrada origen/destino en metros (se truncan a celda); salida vector de celdas reconstruido del mapa de predecesores.
  • Guard max_expansiones por seguridad (caminos imposibles / enormes → []).
  • Determinista (sin aleatoriedad).

Verificación

test_pathfinding.py: conecta origen-destino y es contiguo; determinista; evita especies penalizadas y pendiente alta cuando se suben sus pesos (rodea un muro salvo el hueco); carga de datos reales y camino entre puntos cercanos. 8 tests.

Endpoint POST /trazar (#122) — tipo A

/trazar solo calcula (separación limpia: trazar = pensar, firebreak = aplicar). El cliente envía el vector resultante a POST /firebreak, igual que un trazo manual → reutiliza todo el flujo de cortafuegos sin código nuevo.

// request
{ "origen":  {"x": 799, "z": 696},
  "destino": {"x": 820, "z": 710},
  "pesos":   {"w_pendiente": 50.0}   // opcional; solo los que se quieran cambiar
}
// response
{ "cells": [[799,696], [800,697], ...] }   // [] si no hay camino / fuera de rango

El Pathfinder se cachea (los datos se cargan una vez; no depende del motor).

Godot — herramienta "Auto" (#123)

Herramienta dedicada (botón + tecla A), flujo de 2 clicks: 1. 1er click izq = origen (marcador esfera cian). 2. 2º click izq = destino (marcador naranja) → POST /trazar. 3. Al recibir el vector, Godot lo pinta reutilizando el pincel manual (_pintar_celda_brocha: splat marrón + terraformado) y lo envía a POST /firebreak. Se ve y se aplica igual que un trazo a mano.

Cambiar de herramienta o resetear limpia la selección y los marcadores. La intensidad usa el slider existente.

Nota: se probó un trazado multipunto (N puntos, orden óptimo TSP) (#137) pero se revirtió: el orden óptimo necesita A* entre todos los pares (O(N²)) y tardaba demasiado para uso interactivo. Se mantiene el de 2 puntos.
