Objetivo

Crear la capa de orquestación entre el motor C++ y Godot. La API gestiona el estado de la simulación en memoria y expone endpoints HTTP.

Decisiones arquitectónicas

  1. API stateful — guarda el estado completo del grid en memoria entre peticiones. Godot solo llama a /tick; no necesita reenviar el estado completo.

  2. Una simulación a la vez — sin concurrencia, sin gestión de sesiones. Un único estado global en memoria.

  3. Parámetros del motor con valores por defecto — preparados para recibir configuración desde Godot en una iteración futura.

Issues cerradas

#10 — Estructura base

  • Carpeta api/ con main.py, requirements.txt
  • GET / devuelve {"status": "ok"}

#11 — Estado en memoria (state.py)

  • Cuatro funciones: get(), set(), clear(), is_initialized()
  • El estado es el mismo dict JSON que produce el motor — sin transformaciones

#12 — POST /tick y motor.py

  • motor.py lanza el binario C++ como subprocess, le pasa el estado por stdin
  • Endpoint POST /tick: estado de memoria → motor → actualiza memoria → devuelve nuevo estado

#13 — GET /state

  • Devuelve el estado actual sin avanzar tick

#14 — POST /reset

  • Recibe estado inicial en el body, lo carga en memoria
  • Modelo Pydantic EstadoInicial para validación automática

#15 — Tests de integración

  • test_integracion.py: flujo reset → tick → state
  • Verifica que varios ticks consecutivos reducen la coordenada de reacción
