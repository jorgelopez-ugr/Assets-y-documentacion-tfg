Para qué existe este módulo

Implementa la fuerza de flotabilidad: el calor hace subir el gas. Sin buoyancy, las celdas BURNING no tendrían velocidad ascendente y la advección no transportaría el calor hacia arriba — el fuego quedaría estático.

Es el primer paso del tick porque crea las velocidades que la presión y la advección van a usar.

aplicarBuoyancy

void aplicarBuoyancy(Grid& g, float A = 0.1f, float B = 0.5f)

Modifica velocity.z (el eje vertical del fluido) de cada celda en función de su temperatura y densidad:

fuerza = A * temperature - B * density
velocity.z += fuerza

Solo aplica a BURNING y SMOKE. Las celdas FUEL son combustible sólido — no tienen dinámica de fluido. Ver por qué esto es importante: buoyancy-aplicada-a-celdas-FUEL

Los parámetros A y B

A — empuje térmico

Controla cuánto empuja hacia arriba cada grado de temperatura.

A = 0.1, temperature = 800°C  →  fuerza = 80 unidades/tick
A = 0.01, temperature = 800°C →  fuerza = 8 unidades/tick

Con A alto, el gas sube rápido y la advección transporta el calor lejos en cada tick. Si A * temperature * dt > 1, el esquema MacCormack puede volverse inestable (CFL violado). Ver mecanismo-propagacion-via-presion.

B — freno por densidad

Reduce la fuerza cuando el humo es denso. El humo recién generado tiene densidad alta y sube más lento. Con el tiempo la densidad cae (por advección hacia celdas vacías) y el humo ligero sube más rápido.

B = 0.5, density = 0.4  →  freno = 0.2 unidades
B = 0.5, density = 2.0  →  freno = 1.0 unidades (el gas pesado puede bajar)

Valores por defecto y efecto visual

Configuración Efecto
A=0.1, B=0.5 (producción) Columna de fuego que sube rápidamente
A=0.01, B=0.5 (tests) Ascenso lento, numéricamente estable, buena propagación lateral
A=0.0 Sin ascenso — el calor no se mueve verticalmente

Por qué es el primer paso del tick

La buoyancy genera velocidades. La presión las corrige para que sean divergencia-cero. La advección las usa para transportar temperatura. Si la buoyancy se aplicara después de la presión, las nuevas velocidades no habrían pasado por la corrección de incompresibilidad — el fluido se comportaría como compresible.

aplicarBuoyancy  →  (velocidades con divergencia)
resolverPresion  →  (velocidades divergencia-cero)
advectar         →  (temperatura y densidad transportadas con esas velocidades)
