Para qué existe este módulo

Orquesta el bucle de cuadrantes activos. Es la lógica central del rediseño y la única clase con estado dinámico (la lista de cuadrantes activos y la referencia mutable a WorldData).

La clase Simulacion

class Simulacion {
public:
    static constexpr float UMBRAL_ACTIVACION  = 100.0f;  // °C en celda de borde
    static constexpr float TEMP_IGNICION      = 800.0f;  // °C inyectados al ignitar
    static constexpr float K_REACCION_DEFAULT = 0.1f;
    static constexpr float DT_DEFAULT         = 0.5f;

    explicit Simulacion(WorldData& w);

    void activar(uint32_t id);                            // aloca SubGrid + FUEL/AIR
    void ignite(float wx, float wz);                      // BURNING en (wx, wz)
    void tick(float dt = DT_DEFAULT, float k = K_REACCION_DEFAULT);
    void reset();                                         // libera todo

    const std::set<uint32_t>& activos() const;
    const WorldData& world() const;

private:
    WorldData& world_;
    std::set<uint32_t> activos_;

    void propagar_halos();
    void evaluar_activacion();
    void evaluar_desactivacion();
    static void sostener_burning(SubGrid& g);

    static int direccion_x(const Cuadrante& a, const Cuadrante& b);
    static int direccion_z(const Cuadrante& a, const Cuadrante& b);
    static void rango_borde(int dx, int dy,
                            int& xmin, int& xmax,
                            int& ymin, int& ymax);
    static float max_temp_borde(const Cuadrante& a, int dx, int dy);
    static void copiar_borde(Cuadrante& a, Cuadrante& b, int dx, int dy);
};

Métodos públicos

activar(id)

Aloca el SubGrid del cuadrante (50.000 celdas) y marca:

  • En la celda de superficie (cx, cz, y_superficie[idx]): state = FUEL, fuel_coefficient copiado del Cuadrante.
  • El resto: queda como Celda{} por defecto, es decir state = AIR, todos los floats a 0.

Si el cuadrante ya estaba ACTIVO, no hace nada (idempotente). Añade el id al set de activos.

ignite(wx, wz)

Convierte coordenadas mundiales a (cuadrante, celda local) y enciende la celda de superficie correspondiente:

qx = (int)wx / CELDAS_QUAD_X;       // 50
qz = (int)wz / CELDAS_QUAD_Z;       // 50
cx = (int)wx - qx * CELDAS_QUAD_X;
cz = (int)wz - qz * CELDAS_QUAD_Z;

Activa el cuadrante (si no lo estaba), busca y_superficie[cz * 50 + cx], y pone esa celda en BURNING con temperature = 800.0 y reaction = 1.0.

tick(dt, k)

Un tick completo. Ver el flujo paso a paso en Pipeline de Simulación.

void tick(float dt, float k) {
    for (uint32_t id : activos_) {
        Cuadrante& c = world_.cuadrantes[id];
        aplicarBuoyancy(c.grid);
        c.grid = resolverPresion(c.grid);
        c.grid = advectar(c.grid, dt);
        aplicarReaccion(c.grid, k, dt);
        sostener_burning(c.grid);
    }
    propagar_halos();
    evaluar_desactivacion();   // primero: libera muertos
    evaluar_activacion();      // después: enciende vivos
}

reset()

Marca todos los cuadrantes activos como INACTIVO, libera sus SubGrids (cells.clear()) y vacía el set de activos.

Métodos privados

sostener_burning(grid) — fuente de calor sostenida

for (Celda& c : grid.cells) {
    if (c.state == EstadoCelda::BURNING && c.temperature < TEMP_IGNICION) {
        c.temperature = TEMP_IGNICION;
    }
}

Sin esto, una celda BURNING termina con temperatura cero al cabo de unos ticks porque la flotabilidad y la advección la drenan. La reinyección modela que una celda ardiendo es un emisor sostenido mientras tenga combustible.

propagar_halos() — intercambio de bordes

Para cada par de cuadrantes activos adyacentes (procesado una sola vez por pareja con if (nid <= id) continue), llama a copiar_borde(a, b, dx, dy) si la dirección es estrictamente cardinal (no diagonal).

copiar_borde promedia simétricamente temperature, density y reaction entre la columna borde de A y la columna borde de B.

evaluar_activacion() — la cascada

Para cada cuadrante activo, mira sus vecinos inactivos. Calcula la temperatura máxima en el borde de A que mira hacia el vecino. Si supera UMBRAL_ACTIVACION = 100 °C, llama a activar(vecino).

Resultado: el fuego se propaga a un nuevo cuadrante en cuanto la temperatura del borde de un cuadrante activo supera el umbral.

evaluar_desactivacion() — libera muertos

for (uint32_t id : activos_) {
    bool actividad = false;
    for (const Celda& cell : c.grid.cells) {
        if (cell.state == EstadoCelda::BURNING ||
            cell.temperature > UMBRAL_ACTIVACION) {
            actividad = true; break;
        }
    }
    if (!actividad) /* desactivar */;
}

La condición de "actividad" incluye temperatura > UMBRAL_ACTIVACION además de BURNING. Esto es esencial: un cuadrante recién activado por evaluar_activacion todavía no tiene celdas BURNING, solo FUEL con calor recibido por halos. Si solo se mirase BURNING, se desactivaría inmediatamente y al re-activarse el SubGrid se reinicializaría perdiendo los halos. Ver: evaluar-desactivacion-pisaba-halos.

Orden importa: desactivar antes de activar

evaluar_desactivacion();   // primero
evaluar_activacion();      // después

Si fueran al revés, un vecino recién activado (sin BURNING todavía) se desactivaría en el mismo tick, abortando la propagación.

Con este orden, cualquier cuadrante recién activado en este tick sobrevive hasta el siguiente, dándole oportunidad a:

  • recibir halos de su vecino activo,
  • ignitar sus FUEL al recibir calor > 100 °C en el siguiente aplicarReaccion.

Tests del módulo

engine/test_simulacion.cpp cubre 18 asserts:

  • activar inicializa correctamente FUEL/AIR según y_superficie.
  • activar es idempotente.
  • ignite deja la celda objetivo en BURNING con TEMP_IGNICION.
  • Propagación a vecino en 1 tick: ignición en el borde derecho de un cuadrante hace que evaluar_activacion active al vecino derecho.
  • evaluar_desactivacion libera cuadrantes sin actividad y borra su grid.
  • reset deja el estado limpio.
