Peculiaridad: el campo `reaction` tiene semántica distinta según el estado de la celda

← Inicio · Cuadrante y Celda · MVP 4 — Propagación real de fuego

MVP afectado: MVP 4 Archivos involucrados: engine/celda.h · engine/reaccion.h · tools/build_initial_state.py

El campo reaction y su significado original

El campo reaction (coordenada de reacción Y) fue diseñado para celdas BURNING:

Y = 1.0  →  recién encendida, combustible completo
Y = 0.0  →  combustible agotado, la celda pasa a SMOKE

Cada tick, aplicarReaccion lo reduce: Y -= k * dt * fuel_coefficient. Cuando llega a 0, la celda transiciona a SMOKE.

El error inicial al crear las celdas FUEL

Al añadir el estado FUEL en MVP 4, build_initial_state.py generaba las celdas de superficie con "reaction": 1.0 — el mismo valor que una celda recién encendida:

# INCORRECTO
cells.append({
    "state": "FUEL",
    "reaction": 1.0,   # ← solo tiene sentido para BURNING
    ...
})

Esto era semánticamente incorrecto. Una celda FUEL no está ardiendo, no tiene "combustible restante que se consume" — tiene combustible que todavía no ha empezado a consumirse. El valor 1.0 solo tiene significado en el contexto de BURNING.

La corrección

Las celdas FUEL generadas por build_initial_state.py usan "reaction": 0.0:

# CORRECTO
cells.append({
    "state": "FUEL",
    "reaction": 0.0,   # sin proceso de combustión activo
    ...
})

Cuando una celda FUEL supera el umbral de ignición y pasa a BURNING, aplicarReaccion la inicializa explícitamente:

if (c.state == EstadoCelda::FUEL && c.temperature > UMBRAL_IGNICION) {
    c.state    = EstadoCelda::BURNING;
    c.reaction = 1.0f;   // ← se asigna aquí, no se hereda de FUEL
}

La transición es la fuente autoritativa del valor inicial de reaction. Lo que contenga el campo en estado FUEL es irrelevante.

Por qué no causó un bug visible (pero igual era incorrecto)

aplicarReaccion itera sobre el grid pero solo actúa en celdas BURNING:

if (c.state == EstadoCelda::FUEL && c.temperature > UMBRAL_IGNICION) { ... }
if (c.state != EstadoCelda::BURNING) continue;   // ← FUEL con reaction=1.0 se salta aquí

Una celda FUEL con reaction=1.0 nunca pierde reaction (porque el código sale en continue antes del decremento). El valor erróneo persistía en el JSON pero no afectaba al comportamiento.

Sin embargo, tener reaction=1.0 en celdas FUEL podría confundir a cualquier sistema que lea el campo sin mirar primero el estado — por ejemplo, una futura visualización que usara reaction para colorear la intensidad del fuego.

Regla para futuras celdas de nuevos estados

Cada estado tiene una semántica propia para los campos de la celda:

Estado reaction temperature
EMPTY irrelevante (0.0) irrelevante (0.0)
FUEL 0.0 (sin combustión) 20.0 (temperatura ambiente)
BURNING 1.0 al encenderse, decrece hasta 0 alta (800°C típico)
SMOKE 0.0 (agotado) variable (hereda de advección)
