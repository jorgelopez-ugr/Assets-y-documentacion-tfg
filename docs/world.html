Para qué existe este módulo

Define WorldData, la estructura raíz que carga data/world.dat (binario generado offline por tools/build_world.py) y queda como fuente de verdad estática durante toda la sesión.

WorldData

struct WorldData {
    static constexpr char MAGIC[4] = {'W', 'L', 'D', '1'};

    uint16_t width_metros = 0;             // 1598
    uint16_t depth_metros = 0;             // 1393
    uint8_t  cuadrantes_x = 0;             // 32
    uint8_t  cuadrantes_z = 0;             // 28
    uint8_t  celdas_por_cuadrante_x = 0;   // 50
    uint8_t  celdas_por_cuadrante_z = 0;   // 50
    uint8_t  niveles_y = 0;                // 20

    std::vector<Cuadrante> cuadrantes;     // 896

    static WorldData cargar(const std::string& ruta);
};

cargar valida el magic WLD1, comprueba que las dimensiones de cuadrante coinciden con las constantes del motor (CELDAS_QUAD_X/Z, NIVELES_Y), parsea los 896 cuadrantes y verifica que no quedan bytes sobrantes.

Formato binario de world.dat

Little-endian, sin padding entre campos. Debe coincidir exactamente con lo que escribe tools/build_world.py.

Header (13 bytes)

4s   magic = "WLD1"
u16  width_metros
u16  depth_metros
u8   cuadrantes_x
u8   cuadrantes_z
u8   celdas_por_cuadrante_x
u8   celdas_por_cuadrante_z
u8   niveles_y

Por cada cuadrante (en orden id ascendente)

u32  id
f32  origen.x
f32  origen.z
f32  pendiente_media
f32  pendiente_dir.x
f32  pendiente_dir.z
u8   num_vecinos
u32  vecinos[num_vecinos]

Por cada celda de superficie (cz outer, cx inner — 2.500 celdas):
  u8   y_superficie
  f32  fuel_coefficient

Tamaño total de world.dat: ~11 MB.

Cómo se usa

#include "world.h"

int main(int argc, char** argv) {
    std::string ruta = argv[1];
    WorldData world = WorldData::cargar(ruta);
    // world.cuadrantes[i] está poblado con datos estáticos.
    // grid (dinámico) sigue vacío hasta que Simulacion lo active.
}

WorldData::cargar lanza std::runtime_error si:

  • el fichero no se puede abrir,
  • el magic no coincide (WLD1),
  • las dimensiones de cuadrante no coinciden con las del binario compilado,
  • hay bytes sobrantes al final (formato corrompido o versión distinta).

Implementación de la lectura

world.h es header-only. cargar lee campo a campo con std::ifstream y una función helper:

template <typename T>
T leer_valor(std::istream& in) {
    T v{};
    leer(in, &v, sizeof(T));
    return v;
}

No usamos read directo de la estructura porque el padding y el alineamiento de C++ no coinciden necesariamente con el formato binario. Leer campo a campo garantiza portabilidad entre compiladores.

Relación con el script offline

tools/build_world.py y engine/world.h son las dos caras del mismo formato. Cualquier cambio en uno debe replicarse en el otro:

  • las constantes (MAGIC, niveles_y, cuadrantes_x/z, etc.)
  • el orden y tipos exactos de los campos
  • el orden de iteración de las celdas (cz outer, cx inner)

Tests cruzados:

  • tools/test_build_world.py valida lo que escribe el script.
  • engine/test_world_loader.cpp valida lo que lee el motor del fichero generado por el script.

Si los dos pasan, formato y código están sincronizados.
