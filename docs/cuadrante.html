Archivo fuente: engine/cuadrante.h Usado en: world.h (carga), simulacion.h (gestión activa)

Para qué existe este módulo

Define las dos estructuras del rediseño que sustituyen al Grid monolítico anterior:

  • SubGrid: el grid 3D fijo de un cuadrante, con la misma interfaz que el Grid antiguo (sizeX/sizeY/sizeZ, get, isValid) para que las cabeceras de física no necesiten cambios.
  • Cuadrante: la unidad de procesamiento. Contiene los datos estáticos del terreno (origen, vecinos, pendiente, fuel) y un SubGrid dinámico que solo se aloca al activarse.

También define:

  • EstadoCuadrante: INACTIVO, ACTIVO, HALO.
  • Las constantes CELDAS_QUAD_X = 50, CELDAS_QUAD_Z = 50, NIVELES_Y = 20 y los derivados CELDAS_SUPERFICIE = 2.500, CELDAS_VOLUMEN = 50.000.

SubGrid — el grid 3D fijo

struct SubGrid {
    static constexpr int sizeX = CELDAS_QUAD_X;   // 50  — world X horizontal
    static constexpr int sizeY = CELDAS_QUAD_Z;   // 50  — world Z horizontal
    static constexpr int sizeZ = NIVELES_Y;       // 20  — vertical
    static constexpr std::size_t total = sizeX * sizeY * sizeZ;  // 50.000

    std::vector<Celda> cells;                     // vacío hasta inicializar()

    void inicializar();                           // cells.assign(total, Celda{})
    bool inicializado() const;                    // cells.size() == total

    Celda& get(int x, int y, int z);
    bool isValid(int x, int y, int z) const;
};

Por qué empieza vacío

896 cuadrantes × 50.000 celdas × 36 bytes ≈ 1.6 GB si se alocara todo por defecto. Inaceptable.

SubGrid::cells empieza como std::vector<Celda> vacío. Solo se aloca (inicializar()) cuando el cuadrante pasa a ACTIVO. En evaluar_desactivacion se vuelve a vaciar (cells.clear()) para liberar memoria.

Indexado lineal

index(x, y, z) = x + sizeX * (y + sizeY * z);

X es el eje que varía más rápido — los bucles del motor iteran for z → for y → for x para acceso secuencial a memoria.

Por qué Z es el eje vertical

Las cabeceras heredadas (buoyancy, presion, adveccion) ya estaban escritas asumiendo velocity.z como dirección vertical. Mantener esa convención permite que las físicas no necesiten ningún cambio en el rediseño. La incongruencia con la nomenclatura del documento de diseño (niveles_y = 20) es deuda visible y aceptada.

EstadoCuadrante

enum class EstadoCuadrante {
    INACTIVO,   // SubGrid no alocado, cuadrante no se procesa
    ACTIVO,     // física completa cada tick
    HALO        // procesado solo en bordes (uso futuro)
};

Transiciones gestionadas por Simulacion:

INACTIVO  ──(activar / evaluar_activacion)──▶  ACTIVO
ACTIVO    ──(evaluar_desactivacion)──────────▶  INACTIVO

HALO no se usa todavía. Está reservado para una futura optimización donde solo se procese una franja de celdas borde de los cuadrantes vecinos a un ACTIVO, sin alocar el SubGrid completo.

Cuadrante — la unidad de procesamiento

struct Cuadrante {
    uint32_t id = 0;                          // 0 .. 895
    Vec2     origen;                          // (qx*50, qz*50) en metros
    float    pendiente_media = 0.0f;
    Vec2     pendiente_dir;                   // gradiente medio del DEM (XZ)
    std::vector<uint32_t> vecinos;            // ids adyacentes (Moore)

    EstadoCuadrante estado = EstadoCuadrante::INACTIVO;

    // Datos estáticos cargados de world.dat
    std::array<uint8_t, CELDAS_SUPERFICIE> y_superficie{};
    std::array<float,   CELDAS_SUPERFICIE> fuel_coefficient{};

    // Datos dinámicos
    SubGrid grid;
};

Datos estáticos

Cargados una sola vez de world.dat en WorldData::cargar y nunca más modificados:

  • id: índice global, qz * cuadrantes_x + qx.
  • origen: coordenada mundial del origen del cuadrante en metros.
  • vecinos: ids de cuadrantes adyacentes según vecindad Moore (8 para interior, 5 para borde, 3 para esquina).
  • pendiente_media y pendiente_dir: calculadas con np.gradient sobre el bloque 50×50 del DEM.
  • y_superficie[idx]: nivel vertical (0..19) de la celda de superficie en la columna mundial (cx, cz), indexado por idx = cz * 50 + cx.
  • fuel_coefficient[idx]: combustibilidad de esa celda. 1.0 dentro del DEM, 0.0 en celdas fuera del DEM (cuadrantes de borde recortados).

Datos dinámicos

  • estado: el EstadoCuadrante actual.
  • grid: el SubGrid con las celdas físicas del tick actual. Vacío mientras el cuadrante está INACTIVO.

Simulacion::activar(id) aloca el grid, marca FUEL en la celda de superficie de cada columna y deja el resto como AIR (Celda por defecto).

Ejemplo: encontrar la celda de superficie en (cx, cz)

const Cuadrante& c = world.cuadrantes[id];
int cx = 25, cz = 25;
int idx = cz * CELDAS_QUAD_X + cx;
int v_sup = c.y_superficie[idx];           // nivel vertical de la superficie
float fc  = c.fuel_coefficient[idx];       // combustibilidad

// Tras activar:
Celda& celda_superficie = c.grid.get(cx, cz, v_sup);
// celda_superficie.state = FUEL
// celda_superficie.fuel_coefficient = fc
