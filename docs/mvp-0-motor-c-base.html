Objetivo

Construir el núcleo de simulación de fuego: un programa C++ que recibe el estado de un grid 3D en JSON, procesa un tick de simulación física volumétrica y devuelve el siguiente estado en JSON.

Qué se construyó

Estructura de datos

  • Vec3 — vector 3D de floats (x, y, z) para velocidades y fuerzas
  • Celda — unidad mínima de simulación: velocidad, presión, densidad, temperatura, coordenada de reacción y estado (EMPTY, BURNING, SMOKE)
  • Grid — contenedor 3D plano de celdas con acceso por coordenadas y validación de límites

Ver detalle: Cuadrante y Celda

Sistemas físicos — en orden de ejecución por tick

  1. Buoyancy (buoyancy.h) — las celdas calientes suben, las densas bajan. f = (A·temp - B·density)·z
  2. Presión (presion.h) — solver de Jacobi para flujo incompresible. Calcula divergencia, itera la ecuación de Poisson y corrige velocidades.
  3. Advección (adveccion.h) — esquema MacCormack de 3 fases: forward, backward y corrección. Transporta temperatura, densidad y reacción.
  4. Reacción (reaccion.h) — consume combustible: reaction -= k·dt. Cuando reaction ≤ 0, la celda pasa de BURNING a SMOKE.

Ver detalle: Pipeline de Simulación

Serialización

serializar.h — Grid ↔ JSON con nlohmann/json. Representación sparse: solo se incluyen celdas no-EMPTY, lo que reduce significativamente el payload.

Ver detalle: Protocolo del Motor

Punto de entrada

main.cpp — lee JSON de stdin, ejecuta un tick, escribe JSON a stdout. Acepta parámetros CLI: --k, --dt, --A, --B, --iter.

No sabe nada de red, de Godot ni de la API. Listo para ser envuelto por la capa Python.
