testing-lab · calidad · índice

La suite de tests del proyecto

Todo lo que se verifica, clasificado en tres capas: el motor C++ (unidad + protocolo), la API en Python (unidad + integración + end-to-end) y el testing-lab (la física aislada). Aquí están todos, explicados y por dónde corren.

Motor (C++)

Carga de datos, bucle de simulación y protocolo del binario.
4 runners .sh
bash engine/test_*.sh

API (Python)

Lógica (DEM, A*, estado) + endpoints + end-to-end.
125 tests pytest
cd api && pytest

testing-lab

Un invariante físico por operación, sobre la sonda real.
17 de cordura
.venv/bin/python tests/test_*.py
La pirámide. Muchos tests rápidos y baratos en la base (unidad de C++ y Python), menos de integración (endpoints), y unos pocos end-to-end (caros pero realistas). El testing-lab es una capa transversal: comprueba que la física hace lo que debe — algo que los tests de API no pueden ver porque no miran dentro del solver.

1 · Motor C++ — engine/

Cada runner .sh compila con g++ y ejecuta su prueba contra el world.dat real (sin mocks). Dos son programas C++ con assert; dos son tests funcionales del protocolo del binario.

TestQué verifica
test_world_loader.sh
+ .cpp		 Carga de data/world.dat: dimensiones (1598×1393), nº y vecindad de cuadrantes, y_superficie y fuel por celda.
test_simulacion.sh
+ .cpp		 El bucle de cuadrantes activos (Simulacion::tick): ignición, propagación del fuego y activación/desactivación de cuadrantes.
test_main.sh El protocolo stdin/stdout del binario: IGNITE/TICK/RESET, formato de CHANGED (6 campos), TICK_DONE>0, aparece BURNING, coordenadas en rango, y que RESET vacía el estado.
test_smoke_virtual.sh (#92) Que las celdas de aire calientes se reporten como SMOKE "virtual" al cliente sin tocar el solver.

3 · testing-lab — tests/

Un script por experimento: corre la sonda real y comprueba un invariante físico del CSV (devuelve 0 = OK, 1 = fallo). No miran rendimiento ni protocolo: miran que la física sea correcta.

Comportamiento de cada operación

TestInvariante
test_buoyancy.pyEl empuje vertical es lineal en la temperatura: vz == A·temp.
test_viento.pyEl viento empuja gases (no FUEL), no toca la vertical, y acumula con los ticks.
test_presion.pyEl solver de Poisson converge: el residuo baja de forma monótona.
test_adveccion.pyEl blob de calor se transporta con el flujo (desplazamiento esperado).
test_reaccion.pyIgnición por umbral, quemado (→SMOKE) y enfriamiento exponencial.

Cómo correr cada capa

# Motor (C++) — compila y ejecuta cada runner
bash engine/test_main.sh
bash engine/test_simulacion.sh
bash engine/test_world_loader.sh
bash engine/test_smoke_virtual.sh

# API (Python) — toda la suite pytest
cd api && pytest            # o: python -m pytest -q

# testing-lab — cada script de cordura (0 = OK, 1 = fallo)
for t in testing-lab/tests/test_*.py; do testing-lab/.venv/bin/python "$t"; done
Recuentos en vivo (testing-lab/experiments/tests_overview.py): el número exacto puede variar al añadir tests. Filosofía: tests de unidad rápidos en la base, integración en medio, end-to-end en la cima, y el testing-lab como verificación transversal de la física.
